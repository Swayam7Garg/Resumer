How search results work

This side panel is purely informational – all resume data comes from your FastAPI backend.

The UI expects the /recruiter/search endpoint to return JSON shaped like: 

{
  "items": [/* array of resumes */],
  "total": 42,
  "page": 1,
  "page_size": 5
}

Each resume item should roughly include: id, name, email, skills, role, experience, projects, resume_text.

We highlight matched keywords in names, roles, skills, projects and resume text to speed up scanning.
